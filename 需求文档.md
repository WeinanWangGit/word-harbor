\# 一、项目概述（Project Overview）



\## 1.1 项目名称（暂定）



\*\*Word Harbor / LexiDock（示例）\*\*



\## 1.2 项目目标



构建一个 \*\*以抽卡 + 收藏 + 互动为核心体验的英语单词学习 Web App\*\*，通过游戏化收藏与角色互动机制，降低学习阻力，提高重复接触与长期留存。



\## 1.3 核心原则



\- 抽卡 ≠ 学习 → \*\*互动才算学习\*\*

&nbsp;   

\- 少页面、少系统、可快速上线

&nbsp;   

\- 单人开发可完成（2–4 周 MVP）

&nbsp;   

\- Web 优先，可 PWA 打包 App

&nbsp;   



\## 1.4 目标用户



\- 对二次元 / 抽卡机制有好感的学习者

&nbsp;   

\- 英语中级以下，希望“轻学习”的用户

&nbsp;   

\- 创作者本人 \& 小范围朋友验证

&nbsp;   



---



\# 二、核心玩法设计（Gameplay Loop）



```

打开 App

&nbsp;→ 查看港区（首页）

&nbsp;→ 抽卡获得单词角色

&nbsp;→ 查看卡牌详情

&nbsp;→ 完成一次轻互动

&nbsp;→ 提升掌握度

&nbsp;→ 解锁/点亮图鉴

```



关键闭环：



> \*\*抽 → 看 → 用 → 留 → 再抽\*\*

\## 3.1 页面结构总览

\## 3.2 首页（港区）



\### 功能需求



\- 显示今日剩余抽卡次数

&nbsp;   

\- 随机展示 1–3 个已拥有单词角色

&nbsp;   

\- 角色随机一句“情境台词”

&nbsp;   



\### 组件



\- Header（Logo / 名称）

&nbsp;   

\- 今日抽卡次数 Badge

&nbsp;   

\- 角色展示 Card

&nbsp;   

\- 底部导航（首页 / 抽卡 / 图鉴）

&nbsp;   



---



\## 3.3 抽卡系统（建造）



\### 抽卡规则



\- 每日免费抽卡次数（如 3 次）

&nbsp;   

\- 单卡池（MVP 阶段）

&nbsp;   



\### 稀有度设计



|稀有度|含义|

|---|---|

|⭐|高频基础词|

|⭐⭐|核心表达|

|⭐⭐⭐|地道/习语|

|⭐⭐⭐⭐|高级/情绪化表达|



\### 抽卡流程



1\. 点击抽卡按钮

&nbsp;   

2\. 播放抽卡动画（CSS / Lottie）

&nbsp;   

3\. 弹出结果弹层

&nbsp;   



---



\## 3.4 抽卡结果弹层



\### 功能



\- 展示抽到的卡牌（1 张）

&nbsp;   

\- 显示稀有度光效

&nbsp;   

\- 支持点击进入卡牌详情页

&nbsp;   



---



\## 3.5 单词角色卡牌（核心）



\### 卡牌数据结构（逻辑）

&nbsp;   

\- 稀有度

&nbsp;

&nbsp;

&nbsp;

&nbsp;

---



\## 3.6 卡牌详情页（学习发生页）



\### 页面结构

&nbsp;   

\- 中：情境句（可播放发音）

&nbsp;   

\- 下：轻互动模块

&nbsp;   



\### 互动类型（任选其一）



\- 单选：选择正确使用情境

&nbsp;   

\- 填空：补全关键单词

&nbsp;   

\- 判断：是否合适使用
